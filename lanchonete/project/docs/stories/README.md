# Stories

Esta pasta guarda os markdowns de stories e regras de produto do projeto.

Use estes arquivos como fonte de contexto quando uma mudanca futura mencionar uma story, uma regra operacional ou um fluxo de usuario. Cada story deve registrar:

- objetivo do usuario;
- rotas ou telas envolvidas;
- comportamento esperado;
- criterios de aceite;
- observacoes tecnicas relevantes.
